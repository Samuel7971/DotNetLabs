# 📖 Overview

DotNetLabs is an interactive learning platform designed to help developers understand .NET concepts through practical, executable, and visual demonstrations.

Instead of learning exclusively from documentation, presentations, or isolated code snippets, DotNetLabs provides a collection of hands-on laboratories where developers can explore real-world scenarios, experiment with different implementations, and observe runtime behavior.

The project aims to bridge the gap between theory and practice by transforming technical concepts into interactive learning experiences.

---

# 🎯 Mission

Provide a collaborative platform where developers can learn, teach, and share .NET knowledge through practical laboratories, high-quality documentation, and executable demonstrations.

---

# 🌍 Vision

Become a reference platform for .NET learning by providing an ever-growing collection of interactive laboratories covering software architecture, design patterns, concurrency, security, performance, messaging, databases, and other modern development practices.

---

# 👥 Target Audience

DotNetLabs is intended for:

- .NET Developers
- Software Engineers
- Students
- Technical Communities
- Development Teams
- Professionals preparing for technical interviews
- Developers who enjoy learning by experimenting

# 🎯 Objectives

The project has the following objectives:

- Transform theory into practical experiences.
- Make technical concepts easier to understand.
- Encourage knowledge sharing.
- Standardize technical presentations.
- Provide reusable demonstrations.
- Support developer onboarding.
- Build a long-term technical knowledge base.

---

# 📚 Core Principles

## Learn by Building

Learning happens through experimentation.

Every concept should be demonstrated through executable code whenever possible.

---

## Documentation First

Documentation is part of the product.

Every important feature, architectural decision, and laboratory should be documented.

---

## Architecture Before Implementation

Technical decisions should be discussed and documented before coding.

The project values intentional design over premature implementation.

---

## Consistency

Every laboratory should follow the same structure, making the project easy to navigate and maintain.

---

## Collaboration

Contributing to DotNetLabs should be simple.

New contributors must find clear documentation, standards, and templates that make creating new laboratories straightforward.

---

## Continuous Evolution

DotNetLabs is designed to grow incrementally.

Each new laboratory should improve the platform while preserving consistency and quality.

---

# 🧪 What is a Lab?

A Lab is an isolated learning experience focused on a single technical concept.

Each Lab should contain:

- Concept explanation
- Theoretical foundation
- Practical demonstration
- Executable examples
- Source code
- Best practices
- Common mistakes
- References
- Automated tests
- Optional learning challenges

---

# 🚫 Non-Goals

DotNetLabs is **not** intended to be:

- A complete .NET course.
- A commercial product.
- A production-ready application template.
- A framework.
- A collection of disconnected code samples.

Its purpose is to provide practical learning experiences.

---

Over time, DotNetLabs should become a centralized platform where developers can explore dozens of .NET topics through interactive demonstrations.

Examples include:

- CancellationToken
- Async Streams
- Channels
- Parallel Programming
- Design Patterns
- DDD
- CQRS
- Authentication & Authorization
- Messaging
- Background Services
- Performance
- Benchmarking
- Cloud Integrations
- Database Patterns
- Testing Strategies

Each topic should provide both theoretical understanding and practical experimentation.

---

# 💬 Philosophy

Reading documentation teaches concepts.

Writing code develops skills.

Experimenting creates understanding.

DotNetLabs exists because the best way to learn software engineering is by building, observing, questioning, and experimenting.

Our goal is not simply to explain how .NET works.

Our goal is to create an environment where developers can experience how .NET works.
